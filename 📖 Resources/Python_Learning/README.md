## Learning Paths by Neurodivergent Type

### ADHD Learning Path

**Strategy:** Sprint method - variety, quick wins, momentum

```
Day 1: Skim main tutorial → Try REPL examples (20 min)
       Focus on: "Try This Now" sections only

Day 2: Start project template Phase 1-2
       Set timer: 25 min Pomodoro
       Goal: Just list files

Day 3: Continue project when interest strikes
       OR: Switch to different examples (variety is OK)

Day 4: Use cheat sheet for experimentation
       Try the one-liners that look interesting

Day 5: Return to project, add one custom feature
       Celebrate small wins
```

**Tips:**
- Keep cheat sheet open for quick dopamine hits
- Use REPL for instant feedback
- Don't force linear progression
- Hyperfocus when it hits, rest when it doesn't

---

### Dyslexia Learning Path

**Strategy:** Visual-first method - diagrams, then code

```
Day 1: Read main tutorial visual sections
       Decision trees, flowcharts, diagrams
       Skip text-heavy parts

Day 2: Study visual quick-start guide
       Understand patterns visually
       Don't memorize, just recognize

Day 3: Open project template
       Read the visual flow diagram
       Understand before coding

Day 4: Type code from template
       See how visual structure becomes code
       Run it, see it work

Day 5: Draw your own project idea
       Find examples that match your drawing
       Build from visual design
```

**Tips:**
- Use syntax highlighting (high contrast)
- Font: OpenDyslexic or Comic Sans is valid
- Text-to-speech for long sections
- Visual structure > memorizing syntax

---

### Autism Learning Path

**Strategy:** Systematic method - thorough, documented, complete

```
Day 1: Read main tutorial start to finish
       Take notes on each section
       Create your own reference doc

Day 2: Work through quick-start guide
       Type every example
       Document results
       Note patterns

Day 3: Project template Phase 1-3
       Complete each phase fully
       Test edge cases
       Document what you learned

Day 4: Project template Phase 4-5
       Add error handling
       Document customizations
       Create your own version

Day 5: Review all materials
       Build comprehensive notes
       Start second project
```

**Tips:**
- Thoroughness is a strength, use it
- Documentation helps pattern recognition
- Consistent environment (same workspace, time)
- Deep dives into concepts are valuable
